# v0.2 -> v0.3 迁移脚本

## 本次变更功能

新增专辑表和歌曲歌手中间表，改用多对多关系：

1. **新增 album 表**：专辑表，存储专辑信息
2. **新增 song_singer 中间表**：歌曲与歌手多对多关系
3. **song 表修改**：
   - 删除 singer_id 字段（原有一对多关系改用中间表）
   - 新增 album_id 字段（关联专辑）

## 表结构变更

### album 表（新建）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int unsigned | 主键 |
| name | varchar(100) | 专辑名 |
| singer_id | int unsigned | 歌手ID |
| pic | varchar(255) | 专辑封面 |
| introduction | varchar(255) | 专辑简介 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

### song_singer 表（新建）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int unsigned | 主键 |
| song_id | int unsigned | 歌曲ID |
| singer_id | int unsigned | 歌手ID |
| create_time | datetime | 创建时间 |

**唯一索引**：(song_id, singer_id) - 防止重复关联

### song 表（修改）
- 删除 singer_id 字段
- 新增 album_id 字段（可空，关联 album 表）

## 执行方式

```bash
mysql -u root -ppassword tp_music < sql/迁移脚本/v0.2-v0.3.sql
```

## 回滚方案

```sql
-- 删除外键和索引
ALTER TABLE song DROP FOREIGN KEY fk_song_album;
ALTER TABLE song DROP KEY idx_album_id;
ALTER TABLE song DROP COLUMN album_id;

-- 恢复 singer_id
ALTER TABLE song ADD COLUMN singer_id int unsigned DEFAULT NULL;
ALTER TABLE song ADD KEY idx_singer_id (singer_id);
ALTER TABLE song ADD CONSTRAINT fk_song_singer FOREIGN KEY (singer_id) REFERENCES singer(id) ON DELETE RESTRICT ON UPDATE CASCADE;

-- 从中间表恢复数据
INSERT INTO song (id, singer_id)
SELECT ss.song_id, ss.singer_id
FROM song_singer ss
ON DUPLICATE KEY UPDATE singer_id = ss.singer_id;

-- 删除中间表和 album 表
DROP TABLE IF EXISTS song_singer;
DROP TABLE IF EXISTS album;
```
