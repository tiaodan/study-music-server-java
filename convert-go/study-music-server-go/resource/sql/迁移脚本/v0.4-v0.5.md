# v0.4 -> v0.5 迁移脚本

## 本次变更功能

重构 song 表 URL 相关字段：

1. **删除** song.url 字段
2. **新增** nas_url（NAS存储路径）
3. **新增** spider_url（爬取原始链接）
4. **新增** spider_url_https（带http/https完整链接）
5. **新增** aws_url（AWS真实链接）
6. **新增** aws_url_temp（AWS临时链接）
7. **新增** full_name（下载时的完整文件名）

## 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| nas_url | varchar(255) | NAS存储路径（如：周杰伦/依然/Rain.mp3） |
| spider_url | varchar(500) | 爬取的原始链接 |
| spider_url_https | varchar(500) | 带http/https的完整链接 |
| aws_url | varchar(500) | AWS真实链接（长期有效） |
| aws_url_temp | varchar(500) | AWS临时链接（有时效性） |
| full_name | varchar(255) | 下载时的完整文件名（如：周杰伦-稻香.mp3） |

## 执行方式

```bash
mysql -u root -ppassword tp_music < sql/迁移脚本/v0.4-v0.5.sql
```

## 回滚方案

```sql
-- 删除新增字段，恢复旧结构
ALTER TABLE song
DROP COLUMN IF EXISTS nas_url,
DROP COLUMN IF EXISTS spider_url,
DROP COLUMN IF EXISTS spider_url_https,
DROP COLUMN IF EXISTS aws_url,
DROP COLUMN IF EXISTS aws_url_temp,
DROP COLUMN IF EXISTS full_name;

ALTER TABLE song ADD COLUMN url VARCHAR(255) NOT NULL AFTER lyric;
UPDATE song s, (SELECT id, nas_url FROM song WHERE nas_url IS NOT NULL) t SET s.url = t.nas_url WHERE s.id = t.id;
```
