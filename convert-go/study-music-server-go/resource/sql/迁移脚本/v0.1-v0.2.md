# v0.1 -> v0.2 迁移脚本

## 本次变更功能

为 song 表添加外键约束，建立 singer 与 song 的关联关系：

1. **删除时限制**：删除 singer 时，如果有关联 song 记录，则拒绝删除（RESTRICT）
2. **更新时级联**：更新 singer.id 时，关联的 song.singer_id 自动同步更新（CASCADE）

## 问题说明

song.singer_id 原始类型为 bigint unsigned，与 singer.id 类型 int unsigned 不一致，需要先修改类型。

## SQL 脚本

```sql
-- 步骤1：修改 singer_id 类型为 int unsigned（与 singer.id 一致）
ALTER TABLE song MODIFY singer_id INT UNSIGNED DEFAULT NULL;

-- 步骤2：为 song 表添加外键约束
-- 关联表：song -> singer
-- 外键字段：song.singer_id -> singer.id
ALTER TABLE song
ADD CONSTRAINT fk_song_singer
FOREIGN KEY (singer_id)
REFERENCES singer(id)
ON UPDATE CASCADE
ON DELETE RESTRICT;
```

## 执行方式

```bash
# 登录 MySQL 后执行
source /path/to/v0.1-v0.2.sql

# 或直接执行
mysql -u用户名 -p 数据库名 < resource/sql/迁移脚本/v0.1-v0.2.sql
```

## 回滚方案

如果需要回滚，执行以下 SQL：

```sql
ALTER TABLE song DROP FOREIGN KEY fk_song_singer;
ALTER TABLE song MODIFY singer_id BIGINT UNSIGNED DEFAULT NULL;
```
