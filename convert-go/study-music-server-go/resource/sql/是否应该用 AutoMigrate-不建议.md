# 是否应该用 AutoMigrate - 不建议

## 大厂做法

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| **迁移文件** | 手动编写 SQL 迁移脚本，按版本号管理 | 生产环境（推荐） |
| **迁移工具** | Flyway、Liquibase、Golang-Migrate | 中大型项目 |
| **Code First** | 每次修改模型自动迁移 | 开发测试环境 |
| **禁止迁移** | 只做增量脚本，数据库变更走审批 | 大厂生产环境 |

## 推荐方案

由于这是学习项目，建议使用**迁移脚本**方式：

1. 创建 `migrations/` 目录
2. 每次数据库变更，手动写 SQL 脚本
3. 脚本按版本号命名，如 `001_add_foreign_key.sql`

**好处：**
- 不会丢失数据
- 可追溯变更历史
- 生产环境安全
- 团队可 Code Review
